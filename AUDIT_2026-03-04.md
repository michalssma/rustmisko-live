### 🔴 P0-1: Duplicitní WON/LOST zápisy v ledgeru

**Impact**: PnL nafouklý, statistiky nepřesné, risk limity ovlivněné  
**Evidence**: 66 WON řádků = jen 49 unikátních bet_id (17 duplikátů). Dnes: 26 WON = 10 unikátních (16 duplikátů!).  
**Root cause**: Claim poll loop (cca 60s interval) zapíše WON pokaždé, když najde settled bet — **ale neodstraní bet z `active_bets` mezi iteracemi**. Bet je přidán do `bets_to_remove` a `settled_bet_ids`, ale `active_bets.retain()` se provede až PO celé iteraci (řádek 6931). Navíc PATH B (settled-bet detection) může zapsat WON i po PATH A zápisu, protože `settled_bet_ids` je per-iteration `HashSet` — funguje v rámci jedné iterace, ale v další iteraci se cyklus znovu prochází `active_bets` které ještě nebyly odstraněny.

**Fix** (řádek 6524):
```rust
// PŘED pollováním bet NUTNO zkontrolovat globální settled set
if settled_bet_ids.contains(&bet.bet_id) {
    continue;  // ← tohle JE v kódu, ALE settled_bet_ids se resetuje?
}
```
⚠️ `settled_bet_ids` žije celou session (řádek 4139), takže guard existuje, ale **nefunguje proti race condition** kdy PATH A zapíše WON a za <60s další poll cycle najde ten samý bet přes PATH B (jiné API endpoint vrátí "settled" stav).

**Proposed fix**: Přidat `if settled_bet_ids.contains(&bet.bet_id) { continue; }` na ZAČÁTEK PATH B bloku (řádek ~6660), a/nebo přidat dedup guard přímo do `ledger_write()`:

### 🔴 P0-4: Anomaly stake systém NEUMÍ škálovat na confidence

**Impact**: Perfektní bety jako #582 (CS:GO 11:6) a #587 (tennis 1:0 set) dostávají jen $0.50 místo $2-5  
**Evidence**: Oba dnešní bety = $0.50 = minimum. Důvod viz kód:

```rust
// Řádek 570-585:
const AUTO_BET_ODDS_ANOMALY_STAKE_BASE_USD: f64 = 0.50;  // ← BASE příliš nízký
const AUTO_BET_ODDS_ANOMALY_STAKE_FLOOR: f64 = 0.50;     // ← FLOOR = BASE (nikdy neklesne)
const AUTO_BET_ODDS_ANOMALY_STAKE_CAP: f64 = 1.00;       // ← CAP $1 je směšný

fn anomaly_stake_for_odds(azuro_odds: f64) -> f64 {
    let scale = (1.25 / azuro_odds).powf(1.5);  // ← INVERTOVANÁ logika!
    // Pro odds 1.43: scale = 0.82 → $0.50 * 0.82 = $0.41 → floor na $0.50
    // Pro odds 1.80: scale = 0.58 → $0.50 * 0.58 = $0.29 → floor na $0.50
    // → VŽDY $0.50 pro jakékoli odds > 1.25!
}
```

**Matematika problému**: Formule `(ref/odds)^1.5` klesá s rostoucími odds. Ale protože `ref=1.25` a `floor=$0.50=base`, výsledek je VŽDY floor $0.50 pro jakékoli odds > 1.25. Formule nemá žádný efekt.

Navíc: `ANOMALY_DAILY_LIMIT_MULT = 0.30` → anomaly path dostane jen 30% daily budgetu:
- SOD bankroll $30: effective_daily = min($30, $30 × 0.30) = $9.00
- Anomaly limit: $9.00 × 0.30 = **$2.70/den** → po 5 betech po $0.50 je budget vyčerpaný!

**Proposed fix** — nový Confidence-Based Stake systém:
```rust
const ANOMALY_STAKE_BASE: f64 = 1.50;     // 3× zvýšení
const ANOMALY_STAKE_FLOOR: f64 = 0.50;    // minimum zachováno
const ANOMALY_STAKE_CAP: f64 = 4.00;      // reálný max
const ANOMALY_DAILY_LIMIT_MULT: f64 = 0.60; // 60% budgetu (ne 30%)

fn confidence_stake(odds: f64, disc_pct: f64, live_score_advantage: bool) -> f64 {
    let mut stake = ANOMALY_STAKE_BASE;
    
    // Nízký odds (< 1.50) = vysoká implied probability → zvýšit stake
    if odds < 1.50 { stake *= 1.5; }
    
    // Velká diskrepance (> 25%) = silnější signál → zvýšit stake
    if disc_pct > 25.0 { stake *= 1.3; }
    
    // Live score potvrzuje směr (jako CS 11:6) → maximální confidence
    if live_score_advantage { stake *= 1.5; }
    
    stake.max(ANOMALY_STAKE_FLOOR).min(ANOMALY_STAKE_CAP)
}
// Bet #582: odds 1.43, disc>15%, score 11:6 → $1.50 × 1.5 × 1.0 × 1.5 = $3.38 místo $0.50!
// Bet #587: odds 1.44, disc>15%, 1:0 set  → $1.50 × 1.5 × 1.0 × 1.5 = $3.38 místo $0.50!
```

**Effort**: 2-3 hodiny | **Risk**: Střední | **Rollback**: Restore stávající konstanty

### 3.5 Proč 49% WR + 1.65 avg odds = ztráta?

Breakeven WR pro průměrné odds $o$:

$$WR_{BE} = \frac{1}{o} = \frac{1}{1.65} \approx 60.6\%$$

Aktuální WR = 49% → **deficit 11.6 procentních bodů**. Pro aktuální WR 49% by breakeven odds byly:

$$o_{BE} = \frac{1}{0.49} \approx 2.04$$

Systém sází na odds ~1.65 s winrate 49% → **nemá pozitivní expected value na žádný typ betu**.

## FÁZE 4 — REDESIGN PROPOSALS (Cestou k profitabilitě)

### Proposal 1: Confidence-Based Stake Scaling (CRITICAL)

**Problém**: Bet #582 (CS:GO 11:6 mapa) a #587 (tennis 1:0 set + vedení) = téměř jisté výhry, ale stake $0.50.

**Řešení**: Multi-faktor confidence score:

| Faktor | Multiplikátor | Příklad |
|--------|--------------|---------|
| Odds < 1.50 (high implied prob) | ×1.5 | 1.43 → ano |
| Diskrepance > 25% | ×1.3 | 28% → ano |
| Score advantage potvrzuje | ×1.5 | 11:6 CS → ano |
| Cross-validated (2+ zdroje) | ×1.25 | 2 bookmakeři → ano |

**Příklad: Bet #582** (Fluxo, CS:GO 11:6, odds 1.43):
- Base: $1.50 (zvýšené z $0.50)
- × 1.5 (low odds) = $2.25
- × 1.5 (score advantage 11:6) = $3.38
- **Výsledek: $3.38 místo $0.50 = 6.75× více na nejjistější bet!**

### Proposal 4: Kelly Criterion position sizing

Pro bety s estimated edge (score-confirmed anomálie):

$$f^* = \frac{p \cdot (o - 1) - (1 - p)}{o - 1}$$

Kde $p$ = estimated win probability, $o$ = decimal odds.

Příklad: Bet #582, $p = 0.80$ (CS 11:6), $o = 1.43$:

$$f^* = \frac{0.80 \times 0.43 - 0.20}{0.43} = \frac{0.344 - 0.20}{0.43} = \frac{0.144}{0.43} = 0.335$$

→ Optimální stake = 33.5% bankrollu! Konzervativní ¼ Kelly = 8.4% = **$2.40 z $28.40** místo $0.50.

### 3.2 Steam Roller efekt — kvantifikace


### 4.2 True Probability estimation (MVP)

Nemáme ML model, ale **máme fakta ze skóre**. MVP odhad `true_p`:

```rust
/// MVP True Probability estimate from score state
/// NOT a model — just calibrated heuristics from domain knowledge
fn estimate_true_probability(
    sport: &str,
    score1: i32, score2: i32,        // map/set score
    detailed_score: Option<&str>,      // round/game level
    azuro_odds: f64,                   // market implied probability as reference
    value_side: u8,                    // 1 or 2
) -> (f64, &'static str) {            // (true_p, confidence_label)
    let diff = if value_side == 1 { score1 - score2 } else { score2 - score1 };
    let implied_p = 1.0 / azuro_odds;
    
    match sport {
        "cs2" | "esports" | "valorant" => {
            // CS2: round score from detailed_score
            let round_diff = parse_round_diff(detailed_score);
            match (diff, round_diff) {
                (d, _) if d >= 2  => (0.92, "VERY_STRONG"),  // 2+ maps ahead
                (1, Some(rd)) if rd >= 5 => (0.88, "STRONG"), // 1 map + dominating
                (1, Some(rd)) if rd >= 3 => (0.82, "STRONG"), // 1 map + leading (11:6-ish)
                (1, _)         => (0.70, "MODERATE"),          // 1 map ahead, even rounds
                (0, Some(rd)) if rd >= 5 => (0.65, "MODERATE"), // tied maps, winning big on current
                (0, _)         => (implied_p, "LOW"),           // tied → trust market
                _              => (implied_p * 0.9, "LOW"),     // losing → below market
            }
        }
        "tennis" => {
            match diff {
                d if d >= 2  => (0.90, "VERY_STRONG"),   // 2 sets ahead (near certain)
                1            => (0.70, "MODERATE"),        // 1 set ahead
                0            => (implied_p, "LOW"),        // even sets → trust market
                _            => (implied_p * 0.85, "LOW"), // behind
            }
        }
        "basketball" => {
            // Parse point diff from detailed_score if available
            let point_diff = parse_point_diff(detailed_score);
            match point_diff {
                Some(pd) if pd >= 20 => (0.90, "VERY_STRONG"),
                Some(pd) if pd >= 10 => (0.78, "STRONG"),
                Some(pd) if pd >= 5  => (0.65, "MODERATE"),
                _                     => (implied_p, "LOW"),
            }
        }
        "football" => {
            match diff {
                d if d >= 2 => (0.85, "STRONG"),
                1           => (0.62, "MODERATE"),
                _           => (implied_p, "LOW"),
            }
        }
        _ => (implied_p, "LOW"),
    }
}
```

### 4.3 Režim klasifikace

```rust
#[derive(Debug, Clone, Copy, PartialEq)]
enum BetRegime {
    StrongEdge,      // High confidence, scale up
    FalseFavorite,   // Low confidence despite low odds, minimal or skip
    Quarantine,      // High odds, unproven, learning mode
    NoBet,           // Negative or zero EV, skip entirely
}

fn classify_regime(
    true_p: f64,
    confidence: &str,
    azuro_odds: f64,
) -> BetRegime {
    let implied_p = 1.0 / azuro_odds;
    let edge = true_p - implied_p;
    
    // Hard rule: negative EV → NoBet
    if edge <= 0.0 {
        return BetRegime::NoBet;
    }
    
    // Quarantine: odds >= 2.00, unproven territory
    if azuro_odds >= 2.00 {
        return BetRegime::Quarantine;
    }
    
    // StrongEdge: high true_p + meaningful edge + score evidence
    if true_p >= 0.70 && edge >= 0.05 && (confidence == "STRONG" || confidence == "VERY_STRONG") {
        return BetRegime::StrongEdge;
    }
    
    // FalseFavorite: low odds but no real evidence
    if confidence == "LOW" || edge < 0.03 {
        return BetRegime::FalseFavorite;
    }
    
    // Middle ground: MODERATE confidence with some edge
    if edge >= 0.03 {
        BetRegime::StrongEdge  // allow but with moderate sizing
    } else {
        BetRegime::FalseFavorite
    }
}
```

## SEKCE 5 — STAKE SIZING + RISK LIMITY

### 5.1 Stake sizing vzorec

Frakční Kelly s confidence-weighted cap:

$$f^* = \frac{p(o-1) - (1-p)}{o-1}$$

Kde $p$ = `true_p` z MVP estimátoru, $o$ = decimal Azuro odds.

```rust
fn compute_stake(
    bankroll: f64,
    true_p: f64,
    azuro_odds: f64,
    regime: BetRegime,
) -> (f64, String) {  // (stake, reason)
    let implied_p = 1.0 / azuro_odds;
    let edge = true_p - implied_p;
    
    if edge <= 0.0 {
        return (0.0, "EV<=0, NoBet".into());
    }
    
    let kelly_f = (true_p * (azuro_odds - 1.0) - (1.0 - true_p)) / (azuro_odds - 1.0);
    
    if kelly_f <= 0.0 {
        return (0.0, format!("Kelly<=0 (f={:.3}), NoBet", kelly_f));
    }
    
    let (fraction, floor, cap) = match regime {
        BetRegime::StrongEdge     => (1.0/3.0, 1.00, 5.00),  // 1/3 Kelly, floor $1, cap $5
        BetRegime::FalseFavorite  => (0.0,     0.50, 0.50),   // Fixed $0.50 test stake
        BetRegime::Quarantine     => (0.0,     0.00, 0.50),   // $0 or fixed $0.50 learning
        BetRegime::NoBet          => return (0.0, "NoBet regime".into()),
    };
    
    let raw_stake = if fraction > 0.0 {
        bankroll * kelly_f * fraction
    } else {
        floor
    };
    
    let final_stake = raw_stake.max(floor).min(cap);
    let reason = format!(
        "regime={:?} kelly_f={:.3} raw=${:.2} → ${:.2} (edge={:.1}% trueP={:.0}%)",
        regime, kelly_f, raw_stake, final_stake, edge*100.0, true_p*100.0
    );
    (final_stake, reason)
}
```

### 5.2 Anti-Steam-Roller guardy

```
GUARD 1: Max loss per bet = 15% bankrollu ($4.50 z $30)
          → 1 prohra max smaže 4.50/0.89 = 5 avg výher (ne 3.5 × $3 = $10.50)
          
GUARD 2: Max "wipe ratio" = 3.0
          → stake × 1 (loss) ≤ 3 × avg_recent_win_profit
          → Pokud avg win = $0.89, max stake = $2.67
          → ALE: pro StrongEdge s edge>10% relaxujeme na 5.0
          
GUARD 3: Cumulative exposure cap = 40% bankrollu
          → Max $12 v otevřených betech najednou
          
GUARD 4: Stop-loss per session = $15 ($30 × 50%)
          → Po dosažení: 1h cooldown, pak restart s 50% sizing
```
